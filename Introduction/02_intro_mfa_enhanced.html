Multifactor Authentication

Introduction Level
Back to Level 1

Multifactor authentication adds a second proof of identity so a stolen password alone is not enough to access company systems.

Narrative 01

Password Alone Fails

Stolen passwords are common, so accounts need a second proof.

What it means

A password proves knowledge, but not identity. If an attacker steals that knowledge, they can try to sign in as the user.

Narrative 02

Approve the Right Request

Unexpected MFA prompts are warning signs.

What it means

Approve a prompt only when you personally started the login. If a prompt appears unexpectedly, deny it and report it.

Narrative 03

Use Strong Factors

Stronger factors make account takeover harder.

What it means

MFA can use authenticator apps, one-time codes, push prompts, or hardware security keys. Some methods are stronger than others, but all are better than password-only access.

Narrative 04

Blocked and Reported

Fast reporting helps stop account takeover.

What it means

A suspicious MFA prompt is evidence that a password may already be exposed. Reporting allows the security team to reset credentials and investigate the source.
